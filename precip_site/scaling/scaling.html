Precipitation Intensity Scaling

Deriving IDF Curves via Scale Invariance Across Durations

Commercial Context & Application

Urban flood modeling and sewer design require short-duration precipitation estimates (e.g., 15-minute or 1-hour). However, reliable historical data is often only available at 24-hour daily resolutions. Scale invariance principles allow catastrophe modelers to mathematically bridge this gap, extrapolating sub-hourly extreme rainfall intensities from long-term daily gauge records to build comprehensive Intensity-Duration-Frequency (IDF) curves.

1. Fundamental Theory of Scaling

Scaling theory posits that the statistical properties of precipitation intensity at one duration are related to the properties at another duration via a scale factor. Let $I_d$ be the annual maximum precipitation intensity for duration $d$, and $\lambda$ be a positive scaling factor.

Strict Sense vs. Wide Sense Scaling

There are two primary ways to define this scale invariance:

Strict Sense Scaling (SSS)

Scaling applies to the entire probability distribution. The random variable $I_{\lambda d}$ has the same probability distribution as $\lambda^{-\eta} I_d$, denoted as:

$$I_{\lambda d} \stackrel{d}{=} \lambda^{-\eta} I_d$$

where $\eta$ is the scaling exponent. If this holds, the cumulative distribution functions (CDFs) are related by $F_{I_{\lambda d}}(x) = F_{I_d}(\lambda^\eta x)$.

Because full distributional equality is difficult to prove empirically, hydrologists often rely on a relaxed assumption based on the moments of the distribution.

Wide Sense Scaling (WSS)

Scaling applies only to the moments of the distribution. The expected value (moment) of order $q$ scales as:

$$E[I_{\lambda d}^q] = \lambda^{-K(q)} E[I_d^q]$$

where $K(q)$ is the scale exponent function dependent on the moment order $q$.

2. Simple Scaling vs. Multiscaling

Within Wide Sense Scaling, the behavior of the exponent function $K(q)$ determines whether the precipitation process exhibits simple scaling or multiscaling. This distinction is critical for modeling rare extremes (higher-order moments).

Regime Behavior of $K(q)$ Physical Implication IDF Curve Impact
Simple Scaling Linear: $K(q) = \eta q$ Extremes scale at the exact same rate as the mean precipitation. GEV parameters (location, scale) scale by the same constant $\eta$. Shape parameter $\xi$ is constant across durations.
Multiscaling Non-linear (Concave): $K(q) = c_1 q - c_2 q^2$ Extreme peaks scale differently (often steeper) than moderate rainfall. IDF curves cannot be derived by a single multiplier. Different return periods require different scaling relationships.
Expand for Theory: GEV Parameter Scaling

If simple scaling holds ($I_d \stackrel{d}{=} \lambda^\eta I_{\lambda d}$), and $I_d$ follows a Generalized Extreme Value (GEV) distribution with parameters $(\mu_d, \sigma_d, \xi_d)$, then the parameters for duration $\lambda d$ are analytically defined:

  • Location: $\mu_{\lambda d} = \lambda^{-\eta} \mu_d$
  • Scale: $\sigma_{\lambda d} = \lambda^{-\eta} \sigma_d$
  • Shape: $\xi_{\lambda d} = \xi_d$ (The tail shape is invariant across durations)

This allows a single IDF formulation: $i(d, T) = \frac{A(T)}{d^\eta}$, where $A(T)$ is a function of the return period $T$.

Visualizing the Scaling Exponent $K(q)$

To diagnose the scaling regime, we plot the empirical $K(q)$ against the moment order $q$. A straight line indicates simple scaling; a downward-curving (concave) line indicates multiscaling, which is frequently observed in convective storm systems.

3. Implementation Methodology

To empirically determine scaling properties and fit $K(q)$ from observed gauge data:

  1. Aggregate raw precipitation data into various durations $d$ (e.g., 1h, 2h, 6h, 12h, 24h).
  2. Extract the annual maximum intensity series (AMS) for each duration.
  3. Calculate the non-central moments $E[I_d^q]$ for $q = 1, 2, 3, \dots$ across all durations.
  4. For each $q$, perform a linear regression of $\ln(E[I_d^q])$ against $\ln(d)$. The slope is $-K(q)$.
  5. Plot $-K(q)$ against $q$ to test for linearity (simple scaling) vs. non-linearity (multiscaling).

Code Implementations

Select a language below to see how to estimate the scaling exponent $K(q)$ from multi-duration time series data.

import numpy as np
from scipy.stats import linregress

def calculate_scaling_exponents(intensities, durations, max_q=5):
    """
    intensities: dict mapping duration to an array of Annual Max Intensities
    durations: list of durations in hours
    """
    log_d = np.log(durations)
    K_q = []
    
    for q in range(1, max_q + 1):
        log_moments = []
        for d in durations:
            # E[I^q]: Expected value of the q-th moment
            moment = np.mean(intensities[d] ** q)
            log_moments.append(np.log(moment))
            
        # Linear regression: ln(E[I_d^q]) = -K(q) * ln(d) + C
        slope, intercept, r_val, _, _ = linregress(log_d, log_moments)
        
        # Slope is -K(q), so we negate it to get K(q)
        K_q.append(-slope)
        
    return np.array(K_q)

# Simulated Annual Maxima Data for durations [1, 2, 6, 12, 24] hours
durations_hr = [1, 2, 6, 12, 24]
simulated_ams = {
    1: np.random.gumbel(loc=30, scale=10, size=50),
    2: np.random.gumbel(loc=20, scale=6, size=50),
    6: np.random.gumbel(loc=10, scale=3, size=50),
    12: np.random.gumbel(loc=6, scale=1.8, size=50),
    24: np.random.gumbel(loc=3.5, scale=1.0, size=50)
}

exponents = calculate_scaling_exponents(simulated_ams, durations_hr, max_q=4)

for q, k in enumerate(exponents, start=1):
    print(f"q={q}: K(q) = {k:.3f}")

estimate_Kq <- function(intensities_df, durations, max_q = 5) {
  # intensities_df: rows=years, cols=durations
  
  log_d <- log(durations)
  K_q <- numeric(max_q)
  
  for (q in 1:max_q) {
    log_moments <- numeric(length(durations))
    
    for (i in seq_along(durations)) {
      moment <- mean(intensities_df[[i]]^q)
      log_moments[i] <- log(moment)
    }
    
    # Fit OLS: log(moment) ~ log(duration)
    fit <- lm(log_moments ~ log_d)
    
    # Extract slope (which represents -K(q))
    K_q[q] <- -coef(fit)["log_d"]
  }
  
  return(data.frame(q = 1:max_q, K_q = K_q))
}
