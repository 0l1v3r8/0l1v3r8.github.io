Generalized vs. Ordinary Least Squares

Correcting for Heteroskedasticity and Autocorrelation

Commercial Context & Application

In applied spatial and temporal modeling (e.g., modeling crop yields over time or housing prices across districts), errors are rarely independent and identical. Volatility often scales with the size of the predictor (heteroskedasticity), or residuals correlate across time or space (autocorrelation). Applying Ordinary Least Squares (OLS) in these conditions yields inefficient estimates and overly optimistic standard errors, leading to flawed risk assessments and unreliable confidence intervals.

1. Fundamental Theory

Ordinary Least Squares (OLS) minimizes the sum of squared residuals, producing the Best Linear Unbiased Estimator (BLUE) only if the errors have constant variance ($\sigma^2 I$) and zero covariance. When the error covariance matrix is $\Omega$ (where $\Omega \neq \sigma^2 I$), Generalized Least Squares (GLS) transforms the variables to restore the OLS assumptions on the transformed data.

The OLS and GLS Estimators

Given the linear model $y = X\beta + \epsilon$, where $E[\epsilon] = 0$ and $Var(\epsilon) = \Omega$, the estimators are defined as:

OLS Estimator:

$$\hat{\beta}_{OLS} = (X^T X)^{-1} X^T y$$

GLS Estimator:

$$\hat{\beta}_{GLS} = (X^T \Omega^{-1} X)^{-1} X^T \Omega^{-1} y$$
Expand for Theory: Aitken's Theorem

Aitken's Theorem extends the Gauss-Markov theorem. It proves that if the error variance-covariance matrix $\Omega$ is known, non-singular, and positive definite, the GLS estimator $\hat{\beta}_{GLS}$ is the Best Linear Unbiased Estimator (BLUE).

Since $\Omega$ is positive definite, we can use Cholesky decomposition to find an invertible matrix $P$ such that $\Omega = P P^T$. Multiplying the regression equation by $P^{-1}$ yields:

$$P^{-1}y = P^{-1}X\beta + P^{-1}\epsilon$$

The variance of the transformed errors is $P^{-1} \Omega (P^{-1})^T = I$. Applying OLS to this transformed equation yields the GLS estimator.

2. Methodological Differences

While OLS treats every observation equally, GLS applies a weighting matrix ($\Omega^{-1}$). Observations with higher variance are down-weighted, and observations that are highly correlated with others provide less independent leverage.

Feature Ordinary Least Squares (OLS) Generalized Least Squares (GLS)
Error Assumption Homoskedastic and Independent ($\sigma^2 I$) Heteroskedastic and/or Autocorrelated ($\Omega$)
Observation Weighting Equal weight for all data points Weights based on inverse of variance/covariance
Consequence of Misuse Valid coefficients, but biased standard errors Requires estimating $\Omega$; poor estimate degrades fit
Common Subsets N/A Weighted Least Squares (WLS), Feasible GLS (FGLS)

Visualizing the Impact: Heteroskedasticity

In the simulation below, the variance of the errors increases as $X$ increases (heteroskedasticity). An outlier at high $X$ pulls the OLS line toward it. However, a specific form of GLS—Weighted Least Squares (WLS)—recognizes the high variance at larger $X$ values and gives those points less influence, keeping the regression line truer to the underlying trend.

The "Feasible" Catch

In reality, the true covariance matrix $\Omega$ is never known and must be estimated from the data itself. This is called Feasible GLS (FGLS). If the structural assumption of $\Omega$ is wrong, FGLS can perform worse than OLS with robust standard errors.

3. Implementation Methodology

To implement GLS to handle heteroskedasticity or autocorrelation, the workflow is:

  1. Fit a standard OLS model and extract the residuals.
  2. Analyze residuals to determine the structure of $\Omega$ (e.g., variance proportional to $x^2$, or AR(1) temporal correlation).
  3. Construct the weights or the inverse covariance matrix.
  4. Refit the model using the GLS/WLS algorithm.

Code Implementations

Select a language below to see how to implement GLS (specifically Weighted Least Squares) to correct for heteroskedasticity.

import numpy as np
import statsmodels.api as sm

# 1. Generate Synthetic Heteroskedastic Data
np.random.seed(42)
x = np.linspace(1, 20, 50)
X = sm.add_constant(x)

# True parameters: Intercept=5, Slope=2
# Error variance grows proportionally with x
errors = np.random.normal(0, x * 0.5) 
y = np.dot(X, [5, 2]) + errors

# 2. Fit Ordinary Least Squares (OLS)
ols_model = sm.OLS(y, X).fit()

# 3. Fit Generalized Least Squares (WLS specifically)
# Weight is inversely proportional to the variance (x^2)
weights = 1.0 / (x ** 2)
gls_model = sm.WLS(y, X, weights=weights).fit()

print("OLS standard errors:", ols_model.bse)
print("GLS standard errors:", gls_model.bse)
OLS standard errors: [1.259 0.104] GLS standard errors: [0.384 0.089] # Note the tighter standard errors in GLS due to efficiency gains.
library(nlme)

# 1. Generate Synthetic Heteroskedastic Data
set.seed(42)
x <- seq(1, 20, length.out=50)
errors <- rnorm(50, mean=0, sd=x*0.5)
y <- 5 + 2*x + errors
df <- data.frame(x=x, y=y)

# 2. Fit Ordinary Least Squares (OLS)
ols_model <- lm(y ~ x, data=df)

# 3. Fit Generalized Least Squares (GLS)
# The varPower structure handles variance that increases with x
gls_model <- gls(y ~ x, data=df, weights=varPower(form=~x))

cat("OLS Model Summary:\n")
print(summary(ols_model)$coefficients)
cat("\nGLS Model Summary:\n")
print(summary(gls_model)$tTable)
