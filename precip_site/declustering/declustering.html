Declustering

Identifying Independent Extremes in Time Series Data

Commercial Context & Application

In catastrophe modeling, extreme events like windstorms or floods rarely happen in isolation. A single slow-moving frontal system might cause heavy rainfall across several consecutive days. If we apply Extreme Value Theory (EVT) to every daily exceedance, we violate the foundational assumption that our data points are independent and identically distributed (i.i.d.). Declustering algorithms filter these dependent temporal clusters into single, independent peak events.

1. Fundamental Theory

The Peaks Over Threshold (POT) approach involves selecting a high threshold $u$ and modeling the exceedances above this threshold. According to the Pickands-Balkema-de Haan theorem, as the threshold approaches the absolute maximum, the distribution of these exceedances converges to the Generalized Pareto Distribution (GPD).

The Generalized Pareto Distribution (GPD)

For a random variable $X$ and a high threshold $u$, the conditional excess distribution $F_u(x) = P(X - u \leq x \mid X > u)$ can be approximated by the GPD:

$$G(x) = 1 - \left(1 + \frac{\xi x}{\sigma}\right)^{-1/\xi}$$

Where $x \geq 0$, $\sigma > 0$ is the scale parameter, and $\xi$ is the shape parameter dictating the tail behavior.

Expand for Theory: The Extremal Index ($\theta$)

To quantify the degree of temporal clustering in a time series, extreme value theory relies on the extremal index, denoted as $\theta$. It is defined on the interval $\theta \in (0, 1]$.

  • $\theta = 1$: Indicates perfectly independent extremes (no clustering).
  • $\theta < 1$: Indicates clustering. The reciprocal $1/\theta$ approximately represents the mean cluster size.

For a sequence of dependent random variables, the probability of the maximum being below a threshold $z$ is $P(M_n \leq z) \approx [F(z)]^{n\theta}$, linking the marginal distribution $F(z)$ to the extreme behavior of the clustered process.

2. Identifying Independent Peaks

To satisfy the i.i.d. requirement for fitting the GPD, we must "decluster" the data. The most common algorithmic approach in hydrology and meteorology is the Runs Method, which defines a cluster as a series of exceedances separated by fewer than $r$ consecutive non-exceedances.

Declustering Method Core Mechanism Advantages Disadvantages
Runs Method Groups exceedances separated by fewer than $r$ observations below the threshold $u$. Simple to implement; widely accepted in hydrology. Highly sensitive to the subjective choice of the run length $r$.
Blocks Method Splits data into contiguous temporal blocks and extracts the maximum from each. Avoids defining a run length; easier statistical inference. Can miss multiple true extremes if they fall within the same block.
Physical Tracking Uses meteorological parameters (e.g., storm footprints) to isolate events. Physically robust; eliminates statistical guesswork. Requires complex, multidimensional covariate data.

Key Consideration for Catastrophe Models

Setting the run length $r$ too low will treat a single sprawling storm as multiple independent events, artificially inflating the modeled frequency of extremes. Setting it too high will erase distinct storms that happen to strike in close succession.

3. Implementation Methodology

To implement POT declustering in practice, the process generally follows these discrete steps:

  1. Determine an appropriate threshold $u$ using mean residual life plots or threshold stability plots.
  2. Apply a declustering algorithm (like the Runs Method) with a domain-appropriate gap length $r$.
  3. Extract the maximum value from each identified cluster.
  4. Fit the Generalized Pareto Distribution (GPD) to the remaining independent peaks via Maximum Likelihood Estimation (MLE) or Probability Weighted Moments (PWM).

Select a language below to see how to execute a runs declustering algorithm to isolate independent peaks.

import numpy as np
import pandas as pd

def runs_decluster(series, threshold, run_length):
    """ Declusters a time series using the runs method. """
    is_exceedance = series > threshold
    clusters = []
    current_cluster = []
    gap = 0
    
    for i, (val, exceed) in enumerate(zip(series, is_exceedance)):
        if exceed:
            current_cluster.append((i, val))
            gap = 0
        else:
            gap += 1
            if gap >= run_length and current_cluster:
                clusters.append(current_cluster)
                current_cluster = []
                
    if current_cluster:
        clusters.append(current_cluster)
        
    # Extract the peak (max value) from each cluster
    peaks = [max(cluster, key=lambda x: x[1]) for cluster in clusters]
    return peaks

# Simulated daily precipitation data
rainfall = np.array([5, 12, 45, 52, 48, 10, 2, 0, 60, 65, 12, 4])

# Apply runs method: Threshold=40, run_length=2 days
independent_peaks = runs_decluster(rainfall, threshold=40, run_length=2)

print("Declustered Peaks (Index, Value):")
for idx, val in independent_peaks:
    print(f"Day {idx}: {val}mm")
